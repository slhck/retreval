kappa() click to toggle source

Calculates and returns the Kappa measure for this GoldStandard. It shows to which degree the judges agree in their decisions See: nlp.stanford.edu/IR-book/html/htmledition/assessing-relevance-1.html 

     # File lib/retreval/gold_standard.rb, line 238
238:     def kappa
239:       
240:       # TODO This isn't very pretty, maybe there's a more ruby-esque way to do this?
241:       sum = 0
242:       count = 0
243:       
244:       # A repeated_combination yields all the pairwise combinations of
245:       # users to generate the pairwise kappa statistic. Elements are also
246:       # paired with themselves, so we need to remove those.
247:       @users.values.repeated_combination(2) do |combination|
248:         user1, user2 = combination[0], combination[1]
249:         unless user1 == user2
250:           kappa = pairwise_kappa(user1, user2)
251:           unless kappa.nil?
252:             puts "Kappa for User #{user1.id} and #{user2.id}: #{kappa}" if $verbose
253:             sum += kappa unless kappa.nil?
254:             count += 1
255:           end
256:         end
257:       end
258:       
259:       @kappa = sum / count.to_f
260:       puts "Average pairwise kappa: #{@kappa}" if $verbose
261:       return @kappa
262:     end
load_from_plaintext_file(file) click to toggle source

Parses a plaintext file adhering to the following standard: Every line of text should include a triple that designates the judgement. The symbols should be separated by a tabulator. E.g. 

  my_query        my_document_1     false
  my_query        my_document_2     true

You can also add the user’s ID in the fourth column. 

     # File lib/retreval/gold_standard.rb, line 96

load_from_yaml_file(file) click to toggle source

Parses a YAML file adhering to the following generic standard:

  • “query” denotes the query

  • “documents” these are the documents judged for this query

  • “id” the ID of the document (e.g. its filename, etc.)

  • “judgements” an array of judgements, each one with:

  • “relevant” a boolean value of the judgment (relevant or not)

  • “user” an optional identifier of the user

Example file: 

    * query: 12th air force germany 1957
      documents:
      * id: g5701s.ict21311
        judgements: []

      * id: g5701s.ict21313
        judgements: 
        * relevant: false
          user: 2

pairwise_kappa(user1, user2) click to toggle source

Calculates the pairwise kappa statistic for two users. The two users objects need at least one Judgement in common. Note that the kappa statistic is not really meaningful when there are too little judgements in common! 

     # File lib/retreval/gold_standard.rb, line 270
270:     def pairwise_kappa(user1, user2)
271:       
272:       user1_judgements = user1.judgements.reject { |judgement| not user2.judgements.include?(judgement) }
273:       user2_judgements = user2.judgements.reject { |judgement| not user1.judgements.include?(judgement) }
274:       
275:       total_count = user1_judgements.count
276:       
277:       unless user1_judgements.empty? or user1_judgements.empty?
278:         
279:         positive_agreements = 0     # => when both judges agree positively (relevant)
280:         negative_agreements = 0     # => when both judges agree negatively (nonrelevant)
281:         negative_disagreements = 0  # => when the second judge disagrees by using "nonrelevant"
282:         positive_disagreements = 0  # => when the second judge disagrees by using "relevant"
283:         
284:         for i in 0..(user1_judgements.count-1)
285:           if user1_judgements[i].relevant == true
286:             if user2_judgements[i].relevant == true
287:               positive_agreements += 1
288:             else
289:               negative_disagreements += 1
290:             end
291:           elsif user1_judgements[i].relevant == false
292:             if user2_judgements[i].relevant == false
293:               negative_agreements += 1
294:             else
295:               positive_disagreements += 1
296:             end
297:           end
298:         end
299:         
300:         # The proportion the judges agreed:
301:         p_agreed = (positive_agreements + negative_agreements) / total_count.to_f
302:         
303:         # The pooled marginals:
304:         p_nonrelevant = (positive_disagreements + negative_agreements * 2 + negative_disagreements) / (total_count.to_f * 2)
305:         # This one is the opposite of P(nonrelevant):
306:         # p_relevant = (positive_agreements * 2 + negative_disagreements + positive_disagreements) / (total_count.to_f * 2)
307:         p_relevant = 1 - p_nonrelevant
308:         
309:         # The probability that the judges agreed by chance
310:         p_agreement_by_chance = p_nonrelevant ** 2 + p_relevant ** 2
311:         
312:         
313:         # Finally, the pairwise kappa value
314:         # If there'd be a division by zero, we avoid it and return 0 right away
315:         if p_agreed - p_agreement_by_chance == 0
316:           return 0
317:         # In any other case, the kappa value is correct and we can return it
318:         else
319:           kappa = (p_agreed - p_agreement_by_chance) / (1 - p_agreement_by_chance)
320:           return kappa
321:         end
322:       end
323:       
324:       # If there are no common judgements, there is no kappa value to calculate
325:       return nil
326:     end
